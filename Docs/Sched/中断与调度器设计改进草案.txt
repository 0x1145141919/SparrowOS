中断与调度器设计改进草案
==========================
2026-05-24

=== 一、中断递送框架重构 ===

1.1 hard_interrupt_func_t → interrupt_token_t

  旧:
    typedef void (*hard_interrupt_func_t)(x64_standard_context_v2*, uint8_t, uint32_t);

  新:
    struct interrupt_token_t {
        uint64_t flags;           // bit 0: TOKEN_FLAG_NEED_RESCHED
        uint64_t token_private;   // O(1) 触发源定位（NVMe ctrl×qidx 等）
        void   (*func)(interrupt_token_t* token);
    };

  - handler 无 frame 参数，杜绝在硬件中断上下文跑飞
  - handler 必须返回，框架统一做 EOI + 调度决策
  - 设计文档见 IDT兼容中断递送框架设计.md、实现.md

1.2 两路分流：all_vec_delivery

  all_vec_delivery(frame, vec)
    │
    ├─ soft_interrupt_functions[vec] → 有 frame，可跑飞
    │   ├─ ivec::KTHREAD_CALL (226): yield/exit/wait/sleep/block
    │   ├─ ivec::IPI          (240): AP bringup up/down（跑飞型 IPI）
    │   ├─ ivec::ASM_PANIC    (225): 直接 panic
    │   └─ ivec::SUPRIOUS     (255): 记录返回
    │
    └─ self->tokens[vec].func(&tok) → 无 frame，必须返回
        ├─ LAPIC Timer:    framework EOI + (flags ? sched)
        ├─ i8042 键盘:     framework EOI
        ├─ NVMe CQ:        framework EOI
        ├─ IOMMU 错误:     framework EOI
        ├─ TLB flush IPI:  framework EOI
        └─ ...其他硬件中断

  跑飞型 IPI 和非跑飞型 IPI 分别占不同的向量号，发端投递时选择正确向量。

1.3 alloc_vec_by_apicid

  新增接口，通过 g_gs_by_apicid[apicid] O(1) 查 gs_complex_t，
  免除 caller 维护 processor_id 的负担。

  签名:
    uint8_t alloc_vec_by_apicid(interrupt_token_t* token, uint32_t apicid, KURD_t& kurd);
    extern "C" uint8_t out_interrupt_vec_alloc_by_apicid(...);

1.4 gs_complex_t::tokens[256]

  dispatch[256] → tokens[256]。interrupt_token_t 每个 24 字节。
  布局偏移: [0x0800, 0x1800)。

=== 二、调度器接口优化 ===

2.1 GS slot 直存 task*

  槽位 PROCESSOR_NOW_RUNNING_TID_GS_INDEX → PROCESSOR_NOW_RUNNING_TASK_GS_INDEX
  消除 8 处 task_pool::get_by_tid 读锁竞争。

2.2 kthread_common_save 自取 task*

  签名从 KURD_t save(task*, frame, bool) 简化为 task* save(frame, bool)。
  内部读取 GS 槽位，失败/异常直接 panic，不再返回 KURD_t。
  简化所有 7 个 callers，消除 goto fatals 路径。

2.3 noreturn 注解

  6 个无条件 sched() 飞走的函数标注 [[noreturn]]:
    kthread_yield_true_enter, timer_cpp_enter, kthread_exit_cppenter,
    kthread_self_blocked_cppenter, kthread_sleep_cppenter, block_queue_cppenter

  kthread_wait_cppenter 和 block_if_equal_cppenter 有早期返回路径，不加。

2.4 锁纪律文档

  kthread_interfaces.cpp 顶部注释：
    - scheduler.sched(): 执行流飞走性调用，调用前必须释放所有 RAII 锁
    - scheduler.sleep_tasks_wake(): 内部两次获取/释放 sched_lock

2.5 block_queue_cppenter 修复

  原代码: spinlock_interrupt_about_guard(lock) — 无名临时对象，锁在分号即释放。
  修复: spinlock_interrupt_about_guard g(lock) — 正确持锁至作用域结束。

=== 三、插队机制 ===

3.1 tid_wait_queue 插队 / 排队

  tid_wait_queue 新增:
    bool m_insert_front;
    void set_insert_front(bool b);
    void push(uint64_t tid);  // 根据 m_insert_front 头插或尾插

  i8042 相关队列（analyzed / char_buffer / text_input）全部插队。
  NVMe 使用默认尾插。

3.2 wakeup_thread / insert_ready_task 支持 front

  wakeup_thread(uint64_t tid, bool front_insert=false);
  insert_ready_task(task* ptr, bool front=false);
  front=true 时 push_front 到 ready_queue 头部，而非尾部。
