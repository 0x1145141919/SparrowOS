# 2026-06-02 中断子系统重构日志

## 概述

中断子系统核心重构：从 `ivec` 语义混乱 + 两路分发 + 224 独立 asm 入口 + 符号表扫描 Init
→ 语义清晰的四命名空间 + 三路分发 + 16B trampoline 表 + 算术寻址 Init + FRED 骨架。

涉及 10 个文件的修改。

---

## 变更清单

### 1. 向量号命名空间拆分

**文件**: `fixed_interrupt_vectors.h`

旧: 单 `ivec` 命名空间, 混合 0–31 异常号 + 软中断号 + IPI 号 + `BOTTOM_FOR_SYSTEM_RESERVED_VECS`。
新: 四个语义命名空间:

| 命名空间 | 包含 | 向量范围 |
|---------|------|---------|
| `x86_exceptions` | CPU 异常向量号 | 0–31 |
| `x86_softinterrupt_abi` | 软中断 (int N ABI) | 225–227, 255 |
| `runaway_ipi_vec` | 跑飞型系统 IPI (不返回) | 252–254 |
| `return_ipi_vec` | 返回型系统 IPI | 250–251 |

- `ivec::IPI=240` → 退役 (拆为 6 个固定向量)
- `ivec::LAPIC_ERR=241` → 移除 (语义谎言: 命名固定号但实际动态 alloc)
- `ivec::USER_ABI_ENTER=254` → 移至 `x86_softinterrupt_abi::USER_ABI_ENTER=227`
- `ivec::BOTTOM_FOR_SYSTEM_RESERVED_VECS=224` → 移除 (free pool 边界改为隐式)

### 2. 汇编入口: 224 独立入口 → 16B Trampoline 表

**文件**: `Sysdef_exception_entries.asm`

旧: 224 个 `VEC_DELIVERY_ENTRY` 宏展开, 每个 ~85B, 总计 ~19KB + 224 符号。
新: `vec_demux_table` — 224 个 16B trampoline + `vec_demux_common`:

```
vec_demux_table (224 × 16B = 3.5KB)
  push dword N          ; 向量号入栈, 不破坏 GPR 上下文
  jmp short common
  times 9 nop

vec_demux_common:
  push 15 GPR
  mov rdi, rsp           ; → x64_vec_demux_frame*
  call vec_demux_entry
  pop 15 GPR
  add rsp, 8             ; 弹出 vec
  iretq
```

同时移除了 `EXCEPTION_ENTRY` / `EXCEPTION_ENTRY_WITH_ERRCODE` 宏 (异常入口并入 C)。FRED 入口 (`fred_user_enter`, `fred_supervisor_entry`) 暂保留但空壳。

### 3. 类/文件重命名

**文件**: `x86_vecs_deliver_mgr.h`, `x86_vecs_deliver_mgr.cpp`

| 旧 | 新 | 理由 |
|----|----|------|
| `idt_vec_dispatch_mgr` | `vec_demux` | IDT-only 名 → IDT/FRED 通用 |
| `Init()` | `early_init()` + `late_init()` | 两阶段分离 (boot/MM_READY) |
| `all_vec_delivery(frame, vec)` | `vec_demux_entry(x64_vec_demux_frame*)` | 新签名适应 trampoline 布局 |
| KURD `idt_mgr_events` | `vec_demux_events` | 跟随类名 |
| `modloc_idt_mgr` | `modloc_vec_demux` | 跟随类名 |

### 4. 初始化合并

**文件**: `fixed_interrupt_resources.cpp`

移除 `exceptions_init()` / `exceptions_idt_init()` (约 50 行), 功能并入 `vec_demux::early_init()`。
该文件只保留 `template_idt_apply_region` + `global_idtr`。

新时序:

```
_kernel_Init (asm):
  → vec_demux_early_init()     [extern "C"]
      → 填 template_idt[0..31] (原 exceptions_idt_init)
      → 算术计算 template_idt[32..255] (地址 = &vec_demux_table + (v-32)*16)
      → template_idt_apply_region(0, 255)  一次写入
  → lidt [global_idtr]         ← IDT 256 项全部就绪

kernel_start() (MM_READY):
  → vec_demux::late_init()
      → 清零 tokens 表
      → 初始化 soft_interrupt_functions
      → 初始化 ipi_descrioptors (占位)
```

### 5. 三路分发

**文件**: `x86_vecs_deliver_mgr.cpp`

旧两路:
```
soft_interrupt_functions[vec]? → 跑 (软中断/IPI 共用)
tokens[vec]?                   → 跑 (硬件中断)
```

新三路:
```
soft_interrupt_functions[vec]? → 跑 + UD2 sentinel (纯软中断, int N)
ipi_descrioptors[vec]?         → 跑 + EOI, is_no_return → UD2 (系统 IPI)
tokens[vec]?                   → 跑 + EOI, 检查 resched 标志
```

`ipi_descrioptors[256]` 全局表, 5 个固定向量 (250–254), `alloc_vec` 跳过。

### 6. FRED 骨架

**文件**: `exceptions_handler.cpp`

`fred_common_enter(x64_fred_context*)` 按 type 分发:
- type 0 (extint) → `vec_demux_hw_dispatch` (先 ipi_descrioptors 再 tokens)
- type 2 (NMI) → `nmi_cpp_enter`
- type 3 (exception) → `fred_exceptions_router` (按 vec 路由, 参照 SDM §8.3.2)
- type 4 (soft) → `vec_demux_soft_dispatch` (直接查 soft_interrupt_functions)
- type 7 (syscall) → TODO

`fred_exceptions_router`: 接收原始 FRED 帧, 内部 `filt_frame` 转换 + 
`fred.event_data` 做 #PF 线性地址, `fred.errcode` 做错误码。

### 7. 启动适配

**文件**: `kernel_entry.asm`, `kinit.cpp`

- `kernel_entry.asm`: `exceptions_init` → `vec_demux_early_init`
- `kinit.cpp`: `idt_vec_dispatch_mgr::Init()` → `vec_demux::late_init()`

## 新增结构体

```cpp
// x86_vecs_deliver_mgr.h
struct x64_vec_demux_frame {
    uint64_t rax, rbx, rcx, rdx, rsi, rdi;
    uint64_t r8, r9, r10, r11, r12, r13, r14, r15, rbp;
    uint64_t vec;               // trampoline push dword N 插入在 GPR 与 iretq 之间
    iret_complex_context iret;
};
```

转换函数 (内联, 零开销):
```cpp
frame_to_standard(&ctx, raw_frame);    // 修正 vec 偏移 → x64_standard_context
standard_to_frame(raw_frame, &ctx);    // 写回 handler 修改
```
