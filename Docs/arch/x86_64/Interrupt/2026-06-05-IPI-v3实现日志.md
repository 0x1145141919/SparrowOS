# 工作日志：IPI v3 实现 (2026-06-05 ~ 2026-06-06)

## 背景

将 IPI 机制从 v2（独立向量表 + ipi_descrioptors）迁移到 v3（per-core cmpxchg16b 消息槽）。
涉及 IDT 和 FRED 两条中断路径的统一。

## 改动清单

### GS_complex.h — 新增消息槽

- 新增 `alignas(64) __uint128_t local_ipi_complex` @ 0x40C0
- 新增 `uint64_t padding[6]` 填满 64B 缓存线（适配 UMONITOR 缓存线监视）
- 两个字段位于 `fpu_area` 之后、`stacks_ptr` 之前

### x86_vecs_deliver_mgr.cpp — 核心收发实现

**收端（IDT 路径 `idt_vec_demux_entry`）：**
- `SUPRIOUS_INTERRUPT` (255): 日志 + 返回
- `IPI_HALT` (252): `cli; hlt; __builtin_unreachable()`
- `IPI_RETURNABLE` (253): 读槽 → `fnbox->func(arg)` → 写回 `{1, result}` → EOI
- `IPI_RUNAWAY` (254): 读槽 → cmpxchg 写 `{1, 0}` → EOI → `fnbox.func(arg)` → ud2

**收端（FRED 路径 `fred_vec_demux_hw_dispatch`）：**
- 与 IDT 路径相同，但无 `write_eoi()`（FRED 硬件自动 EOI）
- 移除对已删除的 `ipi_descrioptors` 的引用

**发端：**
- `resolve_target()` — id 到 gs_complex_t* 转换
- `target_x2apicid()` — 从 slot[1] 提取目标的 x2APIC ID
- `make_ipi_icr()` — ICR 构造辅助
- `ret_ipi_send()` — 抢占槽 → 发 RETURNABLE → 轮询 10ms → `{result, status}`
- `fly_ipi_send()` — 抢占槽 → 发 RUNAWAY → 等 lo64=1 → 清 0
- `broadcast_halt()` — ALL_EXCLUDING_SELF 发 IPI_HALT
- `halt_on()` — 定向 IPI_HALT

**轮询机制：**
- `cacheline_wait()` — UMONITOR + UMWAIT(C0.1, EDX:EAX=UINT64_MAX)
- `ipi_wait_lo()` — 环境感知轮询（TCG → pause, 其他 → UMONITOR/UMWAIT）
- `interrupt_guard` 包裹整个临界区（抢占 + 发 + 轮询 + 消费）

**清理：**
- 删除 `ipi_descrioptors` 全局数组及引用
- `alloc_vec` 保护区间从旧命名改为 `ipi_vecs::IPI_HALT..SUPRIOUS_INTERRUPT`
- `cmpxchg16b` extern 声明，替换 cmpxchg16 (6参数) 调用

### fast_get_ids.asm — 处理器 ID 读取纳秒化

新文件 `Processor/fast_get_ids.asm`:
- `fast_get_processor_id()`: `mov rax, [gs:8]; and eax, 0xFFFFFFFF`
- `fast_get_x2apic_id()`: `mov rax, [gs:8]; shr rax, 32`
- 从 `intel_cpuid.cpp` 删除了同名 C++ 内联汇编实现，链接器自动选 NASM 版本

### APs_bringup.cpp — slot[1] 写入双重编码

第二遍扫描重写：
- 删除 `SecondPassCtx` 结构体 + `walk_processors` 回调包装
- BSP: `slot[1] = self_x2apicid << 32 | 0`
- AP: `slot[1] = it->apicid << 32 | pid`
- loop 改用直接 for 循环

### tsc.cpp — IA32_UMWAIT_CONTROL 初始化

- `tsc_regist()` 末尾添加 WAITPKG 探测（CPUID.07h.ECX[5]）
- 不支持 WAITPKG → `tsc_panic_hlt()`（BARE_METAL/KVM 必须支持）
- 公式：`cycles_50us = 50 * FS_per_mius / tsc_fs_per_cycle`
- MSR 值：`[(cycles_50us>>2)<<2 | 1]`
- `apply_umwait_control()` — 供 AP init 序列调用
- `tsc_panic_hlt()` 新增 WAITPKG 状态输出

## 协议细节

### slot 状态机

```
0x0              空闲（sender 可 cmpxchg 抢占）
{fn_lo, arg_hi}  消息待处理（sender 写入成功）
{1, 0}           RUNAWAY 已消费（target 置 lo64=1）
{1, ret_hi}      RETURNABLE 结果就绪（target 写回返回值）
```

### 收端对 slot 的操作

| 操作 | RUNAWAY | RETURNABLE |
|------|---------|------------|
| 读 fn | `fnbox_copy = *slot` | `fnbox_copy = *slot` |
| 写 | `cmpxchg(slot, fnbox_copy→{1,0})` | `cmpxchg(slot, fnbox_copy→{1,ret})` |
| lo64 最终值 | 1 | 1 |

### 发端对 slot 的判断

| 条件 | 含义 |
|------|------|
| `(uint64_t)slot == 0` | 空闲→可抢占 |
| `(uint64_t)slot == func` | target 尚未处理 |
| `(uint64_t)slot != func` | target 已处理（lo64 从 func 变成 1） |

## 已知问题 / 待办

1. **ret=0 的处理**：当前编码 `hi64=ret, lo64=1`，`(uint64_t)slot != func` 判断不受 ret 值影响 ✅
2. **UMONITOR 64B 对齐**：`local_ipi_complex` 是 `alignas(64)`，@0x40C0 是页内偏移 0xC0，64B 对齐 ✅
3. **TCG fallback**：所有 UMONITOR/UMWAIT 路径有 `pause()` 兜底 ✅
4. **IA32_UMWAIT_CONTROL per-core**：BSP init 写入，需确保 AP init 序列调用 `apply_umwait_control()` ⚠️
5. **TLB shootdown 迁移**：当前仍使用旧机制 ⏳
6. **panic halt 迁移**：`other_processors_froze_handler` → `broadcast_halt()` ⏳
